Git 是什麼 ?

Git 是一種分散式版本控制系統。
像是動畫「哆啦Ａ夢」裡的時光機，隨時可以回到過去。
透過這樣的系統，可以清楚的記錄每個檔案是誰在什麼時候加進來、什麼時候被修改或刪除。

HEAD 是什麼東西？

HEAD 是一個指標，指向某一個分支，通常可以把 HEAD 當做「目前所在的分支」。

查詢 HEAD 的移動記錄

合併分支

merge

  • 將當下所在分支與指定分支合併 (兩個由 master 切出去的分支)

注意：當下分支在前，指定分支在後。

  • 當要合併由當下所在分支 master 切出去的分支並「不要使用快轉模式合併」

就會長出「小耳朵」

rebase

  • 將當下所在分支與指定分支合併 (兩個由 master 切出去的分支)

注意：指定分支在前，當下分支在後。

圖片引用 為你自己學 Git - 另一種合併方式（使用 rebase）

感覺像是「把 當下分支 剪下來，然後貼在 指定分支 上面」，
但其實只是因為 原當下分支 已經沒有分支指著它，
看似已經消失不見，事實上它還存在著。
