發表於2023-12-31|Ruby

程式碼區塊 Block

一段 不會被主動執行 的 程式碼。

兩種 Block 的寫法

  • { } 花括號的寫法。
  • do..end 的寫法。

兩種 Block 寫法的差異

  • { } 花括號型的 Block 結合率強

{ } 花括號的優先順序較高

  • do..end 型的 Block 結合率弱

do..end 的優先順序較低，變成前面先跟 p 結合了，造成後面的 Block 就不會被處理了。

Block 不是物件

Block 沒辦法單獨存活

Block 不是參數

透過 yield 來執行 程式碼區塊 Block

name 是參數、"Justin" 是引數，但 Block 不是參數。
say_hello_to 方法內必須要有 yield 才會執行程式碼區塊 Block。

寄生蟲 Block

Block 就像 寄生蟲 一樣，依附在 方法 後面，Block 會不會執行，要看宿主臉色。

yield 會把 控制權 轉讓給 Block 執行 程式碼。

透過 block_given? 檢查 Block 是否存在

  • 如果方法後沒有 Block 但卻 yield 的話，會出現 (LocalJumpError) 的錯誤。
  • 加上 block_given? 防止方法後沒有 Block 產生錯誤訊息。

Block 的回傳值

Block 程式碼結束，自動回傳 Block 最後一行的執行結果。

Block 裡不需要使用 return
