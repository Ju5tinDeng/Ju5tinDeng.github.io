動詞 Prefix 路徑 Controller Action 說明
GET articles /articles ArticlesController index 文章列表

DELETE article /articles/:id ArticlesController destroy 刪除文章

關於 Resources 設定

Resources :單數

遵照 慣例 通常後面會接 複數。
如果後面接 單數，一樣會產生 8 個路徑對應 CRUD 的 7 個 Action 的方法，但對應到的 Controller 會是 單數。

單數 Resource

單數 resource 方法產生的路徑不會有 :id。
只會產生 7 個路徑，檢查會發現少了 index 這個 Action 方法的路徑。
例如：會員系統 「會員不需要知道自己在網站的會員序號」。

調整路徑的數量 Only 與 Except

  • only
  • except

路徑除了只能看 文章總覽 和 各篇文章 ，無法對文章進行 新增、修改、更新和刪除 的其他操作。
only 與 except 兩種寫法相反。

巢狀設計網址

一個使用者擁有多篇文章。
建議 Resources 最多二層就好，避免網址路徑冗長。

簡化贅長的網址路徑 Shallow

或是

在檢視、編輯、更新和刪除文章 操作網址上省略掉 使用者和其序號，讓網址較短、簡單並更清楚。

設定額外獨立的路徑 Collection 與 Member

  • collection

或是

獨立設定檢視「已取消訂單」的路徑。
collection 在路徑上不會產生 :id。

  • member

或是

獨立設定 「確認某訂單」 或是 「取消某訂單」的路徑。
member 在路徑上需帶入 :id 引數。

變更路徑

As

路徑只更改 Helper。

Path

路徑只更改 Path。

Namespace

路徑同時更改 Helper、Path、Controller#Action。

路徑重複

當路徑重複時，前面的路徑會先生效，後面的路徑則是被覆蓋。

View Helper 中 Path 和 Url 的差異

path：產生站內的路徑。

path Prefix_path 路徑
articles + path articles_path /articles
new_article + path new_article_path /articles/new
edit_article + path edit_article_path(5) /articles/5/edit

url：產生含有主機網域名稱的完整網址。

url Prefix_url 網址
articles + url articles_url https://ju5tindeng.github.io/articles
new_article + url new_article_url https://ju5tindeng.github.io/articles/new
edit_article + url edit_article_url(5) https://ju5tindeng.github.io/articles/5/edit
