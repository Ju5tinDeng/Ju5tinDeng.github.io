預設的檔案及資料夾的用途

檔案 ／ 資料夾 用途
+ app / 主要編寫程式的資料夾，包含著應用程式的 Models、Views、Controllers 等。
+ bin / 建構應用程式的 Rails 腳本、環境的設定檔、執行和部署應用程式的腳本。
+ config / 包含應用程式的路徑、資料庫等設定檔。
- database.yml 資料庫的設定檔。
- routes.rb 路徑對照表。
+ db / 資料庫遷移檔案的資料夾 。
+ lib / 應用程式的擴充模組。
+ log / 應用程式存取的記錄檔案。
+ node_modules / 前端套件相關資料夾。
+ public / 唯一對外開放的目錄，不須經過路徑，裡包含著靜態檔案。
+ storage / 主要用於檔案上傳。
+ test / 編寫測試檔。
+ tmp / 暫存檔 (快取、PID、Session 等暫存檔案)。
+ vendor / 非公開套件函式庫，主要放置第三方的程式碼，通常 Rails 應用程式會在這放置第三方的 Gem 套件。
.ruby-version 記錄 Ruby 使用版本。
config.ru 用來啟動應用程式的 Rack 設定檔。
Gemfile 記錄專案使用的套件名稱與版本。
Gemfile.lock 記錄開發專案當下使用的套件的版本。
Rakefile 主要目的是找到並載入可從命令列執行的任務。

VSCode 快速搜尋檔案：command + P

關於版本號

大部分常見的版本號會有三個數字組成，例如：x.y.z。
x 位置是 Major (主要的)，意思是系統大幅度的升級或調整。
y 位置是 Minor (次要的)，代表可能加入些許新功能並向下相容。
z 位置是 Patch (補丁)，意味著針對系統錯誤去做修正及優化並向下相容。

安裝版本為 >= 3.2.1 並 < 3.3。
