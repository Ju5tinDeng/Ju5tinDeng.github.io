Rails - MVC 架構

發表於2024-01-04|Ruby on Rails

MVC 是什麼？

Model：資料邏輯
View：畫面相關
Controller：流程控制

Route：路徑對照表

圖片引用 為你自己學 Git - Model、View、Controller 三分天下

  1. 使用者透過網址進入到網站，根據路徑對照表，找到對應的網頁。
    如果找不到對應的網頁，就會顯示 404 找不到頁面。

  2. 根據使用者輸入的網址和參數，找到相對應 Controller 和 Action。

  3. Controller 可能會有很多個 Action，找到對應的 Action，每個不同的 Action 會決定相對應的事情。
    各種的 Action 其實就是 Controller 裡的各種方法。

  4. 根據 Action 方法的內容，請 Model 幫忙取資料。

  5. Model 是進入資料庫前的虛擬層，本身並不是資料庫。
    Model 會將 ORM 語法 轉換成 SQL 語法 和資料庫溝通。

  6. 透過 SQL 語法 和資料庫取得資料。

  7. Model 將獲得的資料回傳給對應的 Controller 和 Action。

  8. 最後將資料整理和美化，透過 View 呈現給使用者閱讀。

  9. View 其實就是會回傳 HTML 的方法。
